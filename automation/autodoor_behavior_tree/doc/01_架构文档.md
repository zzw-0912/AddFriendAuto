# AutoDoor 行为树系统 — 架构文档

## 1 项目概述

### 1.1 项目定位

AutoDoor 行为树系统是一个独立的可视化行为树编辑与执行框架，面向 Windows 平台的自动化场景（客户端辅助、RPA 流程等）。系统提供完整的图形化编辑器、多种检测/动作节点、脚本录制、OCR 识别、DD 虚拟键盘等能力，并支持标准版与 DD 版两种打包模式。

### 1.2 核心特性

| 特性 | 说明 |
|------|------|
| 可视化编辑器 | 基于 CustomTkinter 的节点式编辑器，支持拖拽、连线、缩放、框选 |
| 行为树引擎 | 独立线程执行，支持启动/暂停/停止/恢复 |
| 24 种内置节点 | 6 种复合节点 + 6 种条件节点 + 12 种动作节点 |
| 黑板系统 | 观察者模式的数据共享机制，节点间解耦通信 |
| 序列化 | JSON/YAML/TXT 多格式持久化，版本化数据结构 |
| 撤销/重做 | Command 模式，支持 100 步历史 |
| DD 虚拟键盘 | DD级硬件模拟输入，绕过大多数输入检测 |
| OCR 集成 | 内嵌 RapidOCR + ONNX Runtime，支持中英文识别 |
| 文本处理 | 文本提取与输入功能，支持 OCR 提取、预设文本、文件输入 |
| 脚本录制 | TXT 脚本录制与回放 |
| 双版打包 | 标准版 (PyAutoGUI) 与DD版 (DD64.dll) |
| 放大镜工具 | 区域/位置选择时显示放大镜，精确定位 |
| 随机范围 | 时长参数支持随机变化，模拟人类操作 |
| 坐标偏移 | 条件节点支持坐标偏移，可视化测量工具 |
| 子树引用 | SubtreeNode 引用外部行为树项目，支持循环检测、黑板隔离、预览展开 |
| 多树并行 | MultiTreeManager 管理多个独立行为树实例，支持共享黑板 |
| 窗口绑定 | 支持绑定目标窗口实现后台运行，自动坐标转换与智能窗口切换 |

### 1.3 技术栈

```
Python 3.10+
├── GUI 框架:    CustomTkinter + Tkinter Canvas
├── 图像处理:    Pillow, OpenCV, imagehash
├── OCR 引擎:    RapidOCR + ONNX Runtime（支持中英文识别）
├── 输入模拟:    PyAutoGUI (标准版) / DD64.dll (DD版)
├── 音频播放:    pygame.mixer
├── 打包工具:    PyInstaller
└── CI/CD:       GitHub Actions
```

## 3 核心组件设计

### 3.1 节点模型 (bt_core/nodes.py)

节点模型是整个系统的核心抽象，采用经典的 Composite 模式设计：

```
Node (ABC)
├── CompositeNode (ABC)
│   ├── SequenceNode     — 顺序执行，全部成功才成功（支持 continue_on_failure、子节点间隔）
│   ├── SelectorNode     — 选择执行，任一成功即成功（支持子节点间隔）
│   ├── ParallelNode     — 并行执行，策略决定成功条件
│   ├── RandomNode       — 随机执行子节点，支持成功策略和完全随机模式
│   ├── StartNode        — 根节点，顺序执行子节点，失败后继续执行后续子节点
│   └── SubtreeNode      — 子树引用节点，加载外部行为树项目执行
├── ConditionNode (ABC)
│   ├── OCRConditionNode     — 文字检测（支持识别方向参数）
│   ├── ImageConditionNode   — 图像模板匹配
│   ├── ColorConditionNode   — 颜色检测
│   ├── NumberConditionNode  — 数字识别比较（支持识别方向参数）
│   ├── VariableConditionNode— 变量值判断（支持常量/变量比较、存在性判断）
│   └── TextExtractNode      — 文本提取（OCR识别后提取文本到黑板）
└── ActionNode (ABC)
    ├── KeyPressNode     — 键盘按键
    ├── MouseClickNode   — 鼠标点击
    ├── MouseMoveNode    — 鼠标移动（支持起点到终点移动、增量移动、拖拽）
    ├── MouseScrollNode  — 鼠标滚动
    ├── DelayNode        — 延时等待
    ├── SetVariableNode  — 设置变量（支持常量赋值/变量赋值、递增、删除）
    ├── ScriptNode       — 执行脚本
    ├── CodeNode         — 执行代码
    ├── AlarmNode        — 播放报警
    ├── TextInputNode    — 文本输入（支持提取值/预设/文件三种输入模式）
    ├── StartTreeNode    — 启动其他行为树
    └── StopTreeNode     — 停止当前或其他行为树
```

**节点核心接口：**

| 方法 | 说明 |
|------|------|
| `tick(context)` | 执行节点逻辑，返回 NodeStatus |
| `reset(reset_counters)` | 重置节点状态，`reset_counters` 控制是否重置计数器 |
| `abort(context)` | 中止节点执行 |
| `add_child(child)` | 添加子节点 |
| `remove_child(child)` | 移除子节点 |
| `to_dict()` | 序列化为字典 |
| `from_dict(data)` | 从字典反序列化 |
| `_execute_children(context)` | 执行子节点（通用实现，位于 Node 基类） |

**关键设计机制：**

| 机制 | 说明 |
|------|------|
| 子节点执行统一 | `_execute_children()` 统一到 Node 基类，ConditionNode 和 ActionNode 不再各自实现 |
| 节点保护 | `_is_protected` 属性防止误删，StartNode 默认受保护 |
| 状态追踪 | `_children_running` 标志追踪子节点执行状态，避免重复执行 |
| 状态管理 | `_reset_for_retry()` / `_reset_for_repeat()` 处理重试/重复时的状态重置 |
| 装饰器模式 | `_execute_with_decorators()` 统一处理 retry/repeat/timeout |
| GUI 状态通知 | ActionNode/ConditionNode 执行子节点前先通知 GUI 节点状态为 SUCCESS |
| 日志输出 | 所有节点在执行成功/失败时通过 LogManager 输出运行日志 |

### 3.2 执行引擎 (bt_core/engine.py)

BehaviorTreeEngine 负责行为树的生命周期管理，采用独立守护线程循环 tick 模式。

**设计要点：**

| 要点 | 说明 |
|------|------|
| 执行模式 | 独立守护线程循环 tick，默认间隔 10ms |
| 状态回调 | `on_status_change` / `on_node_status` 通知外部 |
| 线程安全 | `threading.Lock` 保护启动/停止操作 |
| 可中断等待 | 使用 `threading.Event` 替代 `time.sleep`，支持即时停止 |
| 执行统计 | 集成统计收集器，Debug 环境自动启用 |

**引擎状态机：**

```
    ┌──────────┐
    │  Idle    │ ◄──────────────────────┐
    └────┬─────┘                        │
         │ start()                      │
         ▼                              │
    ┌──────────┐    pause()    ┌────────┤
    │ Running  │──────────────►│ Paused │
    └────┬─────┘◄──────────────└────────┘
         │  resume()
         │
         ├─ stop() ──────► Idle
         │
         └─ 树执行完毕 ───► Completed ──► Idle
```

### 3.3 黑板系统 (bt_core/blackboard.py)

黑板是节点间通信的核心机制，采用观察者模式，支持变量订阅/通知。

**核心接口：**

| 方法 | 说明 |
|------|------|
| `get(key, default)` | 获取变量值 |
| `set(key, value)` | 设置变量值，触发通知 |
| `increment(key, amount)` | 递增数值变量 |
| `delete(key)` | 删除变量 |
| `subscribe(key, callback)` | 订阅变量变化 |
| `unsubscribe(key, callback)` | 取消订阅 |

**内置变量：**

| 变量名 | 说明 |
|--------|------|
| `last_detection_position` | 最近一次检测到的位置坐标 |
| `last_detection_x` / `last_detection_y` | 检测位置坐标分量（自动拆分） |
| `last_number_value` | 最近一次识别的数字值 |
| `last_extracted_text` | 最近一次文本提取的内容 |
| `last_input_text` | 最近一次文本输入的内容 |
| `execution_count` | 执行计数 |

**命名空间黑板 (NamespacedBlackboard)：**

用于子树场景下的数据隔离，支持三种模式：

| 模式 | 说明 | 适用场景 |
|------|------|----------|
| inherit | 共享父级黑板引用 | 默认模式，子树直接读写父树变量 |
| isolated | 独立黑板实例 | 子树拥有完全隔离的数据空间 |
| namespaced | 命名空间前缀隔离 | 变量自动加命名空间前缀，避免冲突 |

**条件节点通用机制：**

- **坐标自动拆分**：`_save_position` 自动将坐标拆分为 x/y 分量写入黑板
- **动态区域检测**：支持固定区域和动态区域（锚点+偏移量）两种模式
- **回退区域**：无 region 时使用上次检测位置附近区域

### 3.4 执行上下文 (bt_core/context.py)

ExecutionContext 封装运行时依赖，采用延迟初始化模式，按需创建截图管理器、输入控制器、OCR 管理器等重量级资源。

**核心职责：**

| 职责 | 说明 |
|------|------|
| 延迟初始化 | 首次调用时创建 ScreenshotManager、InputController、OCRManager |
| 输入引擎路由 | 通过 InputControllerManager 获取键盘/鼠标引擎执行 |
| 窗口绑定 | 支持绑定目标窗口，后台截图，坐标转换，智能窗口切换 |
| 子树调用栈 | 管理子树嵌套调用，限制最大深度 10 层 |
| 帧级缓存 | 同一 tick 内相同 region 的截图结果缓存 |

### 3.5 节点注册中心 (bt_core/registry.py)

NodeRegistry 使用类级别字典管理节点类型映射，支持动态注册与创建。

### 3.6 序列化器 (bt_core/serializer.py)

支持三种格式的序列化/反序列化：

| 格式 | 扩展名 | 说明 |
|------|--------|------|
| JSON | `.json` | 默认格式，完整支持 |
| YAML | `.yaml/.yml` | 需要 PyYAML |
| TEXT | `.txt/.bt` | 纯文本树形表示（仅导出） |

反序列化返回元组 `(root_node, canvas_state, editor_state)`，引擎加载时仅需 `root_node`，编辑器加载时使用完整元组恢复编辑状态。JSON 数据结构采用扁平化节点+连线格式，当前版本为 v2.1（支持子树引用）。

### 4.1 主窗口 (bt_gui/app.py)

BehaviorTreeApp 是 CTk 主窗口，采用 TabView 布局，包含行为树编辑器、脚本录制、设置三个标签页。

**全局快捷键：** Ctrl+Z/Y（撤销/重做）、Ctrl+S（保存）、Ctrl+O/N（打开/新建）、Ctrl+C/V/X/D（复制/粘贴/剪切/快速复制）、Backspace/Delete（删除）。

**焦点保护机制：** Delete/Backspace 在输入框获得焦点或 KeyField 处于按键监听状态时不触发节点删除。

**按键采集统一架构：** 所有按键采集点统一使用 pynput `keyboard.Listener` 采集按键，通过 `key_name_resolver.resolve_key_name()` 统一解析按键名称。

**全局热键管理：** GlobalHotkeyManager 支持组合键识别（修饰键追踪+归一化）、单树快捷键（每个行为树 Tab 独立绑定启动/停止快捷键）、Toggle 逻辑（同一快捷键同时注册启动和停止）。

### 4.2 行为树编辑器 (bt_gui/bt_editor/)

编辑器采用三栏布局：左侧 NodePalette（节点面板）、中间 BehaviorTreeCanvas（画布）、右侧 PropertyPanel（属性面板）。

**画布核心机制：**

| 机制 | 说明 |
|------|------|
| 增量重绘 | 脏标记 + 延迟调度，仅重绘变化区域 |
| Canvas 虚拟化 | 仅渲染视口内可见节点，支持数百节点流畅交互 |
| 拖拽节流 | 16ms/60fps 节流，避免高频重绘 |
| 连接验证 | 节点存在性、重复连接、多父节点、循环连接四重检查 |
| 交互优化 | 延迟加载属性面板、拖拽阈值检测、鼠标按钮状态检查 |

### 4.3 撤销/重做系统 (bt_gui/bt_editor/undo_redo.py)

采用 Command 模式，维护 undo/redo 双栈结构，支持 AddNode、RemoveNode、MoveNode、AddConnection、RemoveConnection、SetProperty、ClearCanvas 等操作类型。粘贴节点使用 AddNodesCommand 确保可撤销，自动定位到视口中心并避免遮挡。

### 4.4 属性面板 (bt_gui/bt_editor/property.py)

PropertyPanel 根据节点类型动态生成表单，支持 text、number、select、bool、region、file、screenshot、key、position、color、offset、region_offset、text_list、random_range、variable_select 等字段类型。

**优化机制：** 同类型节点切换时仅更新字段值（Widget 复用），异步分帧加载避免阻塞事件循环，`hide_if` 支持单条件和 OR 多条件隐藏逻辑。

## 5 基础设施层

### 5.1 输入控制器

系统采用多引擎、键盘/鼠标分离的输入架构，通过配置文件 `input.keyboard_method` 和 `input.mouse_method` 独立选择键盘和鼠标引擎。

| 引擎 | 实现文件 | 输入层级 | 特点 |
|------|----------|----------|------|
| PyAutoGUI | input_controller.py | APPLICATION | 跨平台兼容，易被检测 |
| DD | dd_input.py | DRIVER | 内核驱动级模拟，绕过检测 |
| IbInputSimulator | ib_input.py | DRIVER | Hook+SendInput，支持6种驱动模式 |
| Background | bg_input.py | BACKGROUND | PostMessage 后台操作，窗口不可见时可用 |

**InputControllerManager** 管理双引擎缓存，键盘和鼠标引擎独立选择和创建。

### 5.2 OCR 管理器

OCRManager 封装 RapidOCR + ONNX Runtime，单例模式，提供关键词检测、数字识别、全文识别等接口。内置结果缓存（TTL 1秒，最大 200 条）和缓存命中率统计。

**图像预处理模式：** 所有条件节点共享统一的预处理模式映射（normal/game/adaptive/auto），由 `bt_nodes/conditions/common.py` 定义。

**OCR 优化架构：** RapidOCR 配置优化 + 图像特征分析器 + 智能放大策略 + 自动配置选择器 + 精确关键词定位（中英文字符宽度系数修正）。

### 5.3 截图管理器

ScreenshotManager 封装 PIL.ImageGrab，单例模式，提供全屏截图和区域截图功能。

### 5.4 报警播放器

AlarmPlayer 封装 pygame.mixer，单例模式，支持自定义音效、音量控制、同步/异步播放和停止。

### 5.5 资源管理器

ResourceManager 统一管理应用资源路径（音效、图标等），单例模式，自动区分开发环境和打包环境（`sys._MEIPASS`）解析路径。

### 5.6 放大镜工具

MagnifierWindow 提供屏幕放大镜功能（默认4倍），用于区域选择、位置选择时的精确定位，支持多显示器环境。

### 5.7 识别方向工具

提供识别方向排序功能（SearchDirection 普通类，定义 TOP_LEFT/TOP_RIGHT/BOTTOM_LEFT/BOTTOM_RIGHT 四个方向常量），用于 OCR 相关条件节点在多个匹配结果中选择特定方向的位置。

### 5.8 随机范围辅助工具

提供 `get_random_value()`、`get_random_duration()`、`get_random_interval()` 等工具函数，用于 DelayNode、MouseClickNode、MouseMoveNode、KeyPressNode、SequenceNode 等节点的时长参数随机化。

### 5.9 自动保存与崩溃恢复

编辑器支持自动保存和崩溃恢复功能，存储在用户数据目录（`%APPDATA%/autodoor_behavior_tree/data/backup/`）。启动时自动清理超过7天的崩溃恢复文件，手动保存成功后清理自动保存文件，保留最近3个自动保存备份。

### 5.10 日志管理器

LogManager 提供前端日志和终端日志分离功能。前端日志（log_success/log_failure/log_info）始终显示，终端日志（debug_print）仅在 Debug 环境输出。条件加载机制同时控制执行统计（Debug 使用 StatsCollector，Release 使用 NoOpStatsCollector）。

### 5.11 UI 更新分发器

UIUpdateDispatcher 解决 tkinter 线程安全问题，采用事件驱动 + 定时轮询混合模式。线程安全队列存储待处理任务，自适应间隔轮询（运行时10ms，空闲时100ms），节点状态缓存合并重复更新。轮询在编辑器初始化时启动并持续运行，不因停止操作而中断。

### 5.12 项目工程文件管理

系统提供完整的项目文件管理功能：

| 组件 | 职责 |
|------|------|
| PathResolver | 相对路径与绝对路径转换 |
| ResourceImporter | 资源文件自动导入与分类 |
| ProjectManager | 项目生命周期管理 |
| PackageExporter | 项目打包导出为 ZIP |
| PackageImporter | 从 ZIP 导入项目（含 Zip Slip 防护） |

项目文件夹包含 project.json、tree.json、images/、scripts/、audio/、data/ 等标准目录结构，所有资源使用相对路径引用确保可移植性。保存时自动创建备份（最近5个），导入 ZIP 时验证文件结构并处理同名冲突。

## 10 窗口绑定功能架构

### 10.1 功能概述

窗口绑定功能允许行为树绑定到特定窗口，实现后台运行、后台截图、自动坐标转换、智能窗口切换等能力。

### 10.2 核心组件

| 组件 | 文件 | 职责 |
|------|------|------|
| WindowManager | bt_utils/window_manager.py | 窗口枚举、查找（句柄>PID>标题三级降级）、切换、恢复 |
| WindowCapture | bt_utils/window_capture.py | 后台窗口截图（PrintWindow API） |
| CoordinateConverter | bt_utils/coordinate.py | 屏幕坐标与窗口坐标双向转换 |
| ExecutionContext | bt_core/context.py | 窗口绑定状态管理、智能切换/恢复 |

### 10.3 执行流程

1. **StartNode 初始化**：读取窗口绑定配置，通过三级降级策略（句柄>PID>标题）智能查找窗口
2. **条件节点执行**：使用 WindowCapture 后台截图，坐标从窗口相对坐标转换为屏幕绝对坐标
3. **动作节点执行**：执行前智能切换到绑定窗口，执行后恢复原前台窗口

部分节点（DelayNode、SetVariableNode、AlarmNode、CodeNode、ScriptNode）设置 `SKIP_WINDOW_SWITCH = True` 跳过窗口切换逻辑。

## 12 子树引用与多行为树并行架构

### 12.1 功能概述

子树引用机制允许 SubtreeNode 引用外部行为树项目作为子树执行，支持循环检测、黑板隔离（inherit/isolated/namespaced）、预览展开和文件监控热重载。

多行为树并行功能通过 MultiTreeManager 管理多个独立 TreeInstance，支持共享或隔离黑板数据。

### 12.2 核心组件

| 组件 | 文件 | 职责 |
|------|------|------|
| SubtreeNode | bt_core/nodes.py | 子树引用节点，加载/执行/重置外部行为树 |
| CycleDetector | bt_core/cycle_detector.py | DFS 循环引用检测，路径归一化 |
| NamespacedBlackboard | bt_core/blackboard.py | 命名空间黑板代理，三种隔离模式 |
| TreeInstance | bt_core/tree_instance.py | 行为树实例数据类（name/engine/context/status） |
| MultiTreeManager | bt_core/tree_manager.py | 多树管理器（线程安全，Lock 保护） |
| SubtreeFileWatcher | bt_core/file_watcher.py | 子树文件变化监控与热重载通知 |

### 12.3 子树引用流程

1. SubtreeNode 首次 tick 时加载子树：解析路径 → 查找 tree.json → 循环检测 → 反序列化
2. 创建子树执行上下文（根据黑板模式选择共享/独立/命名空间黑板）
3. 压入子树调用栈，执行子树根节点
4. abort/reset 时弹出调用栈，传播重置

### 12.4 多 Tab 并行架构

在 MultiTreeManager 基础上，GUI 层增加了 Tab 管理、画布切换和持久化支持：

| 组件 | 职责 |
|------|------|
| BehaviorTreeEditor | 代理属性模式，兼容旧代码的 canvas/engine/context 访问 |
| GuiTabManager | 继承 MultiTreeManager，管理 Tab 切换和回调委托 |
| TabBar + TabButton | UI 组件，支持 Tab 切换、运行/停止、关闭 |
| TreeInstance | 扩展 GUI 字段（tab_id/canvas/file_path/command_manager 等） |

**隔离机制：** 每个 Tab 独立拥有画布、引擎、上下文、黑板、命令管理器和自动保存管理器。节点状态通过 tab_id 精确路由到对应画布。

### 12.5 设计决策

| 决策 | 选择 | 原因 |
|------|------|------|
| 子树路径 | 项目文件夹而非 JSON 文件 | 包含完整资源（project.json + tree.json + images/ + scripts/） |
| 循环检测时机 | 加载时检测 | 运行时检测代价高，且可能导致部分执行后失败 |
| 循环检测算法 | DFS + 路径归一化 | O(N) 时间复杂度，覆盖间接循环引用 |
| 黑板默认模式 | inherit | 与原设计兼容，大多数场景不需要隔离 |
| Tab ID 生成 | 类级别递增计数器 | 关闭后重建不会产生重复 ID |
| 画布布局 | place + tkraise() | 所有画布重叠在同一位置，切换时提升到顶层 |
| 多树线程安全 | threading.Lock | 防止并发操作导致的状态不一致 |
