# AutoDoor 行为树系统 — 详细实现方法

## 1 行为树节点类型实现

系统共包含 **24 种节点**：6 种复合节点、6 种条件节点、12 种动作节点。

### 1.1 节点基类 (Node)

所有节点的抽象基类，定义通用接口和默认行为：

```python
class Node(ABC):
    NODE_TYPE = "Node"                    # 类级别节点类型标识
    
    def __init__(self, node_id=None, config=None):
        self.node_id = node_id or uuid4()[:8]   # 唯一标识
        self.config = config or NodeConfig()     # 节点配置
        self.status = NodeStatus.SUCCESS         # 当前状态
        self.children: List[Node] = []           # 子节点列表
        self.parent: Optional[Node] = None       # 父节点引用
        self._tick_count = 0                     # tick计数
    
    @abstractmethod
    def tick(self, context: ExecutionContext) -> NodeStatus:
        """核心方法：执行节点逻辑，返回执行状态"""
        pass
```

**设计要点：**

- 使用 `uuid4()[:8]` 生成短 ID，兼顾唯一性与可读性
- `config` 使用 `NodeConfig` 数据类，采用扁平化结构存储节点参数
- `tick()` 是纯抽象方法，子类必须实现
- `reset(reset_counters=True)` 方法支持控制是否重置计数器，用于 retry/repeat 场景

**节点重置机制：**

| 场景   | 方法                           | 行为                          |
| ---- | ---------------------------- | --------------------------- |
| 完全重置 | `reset(reset_counters=True)` | 重置所有状态和计数器                  |
| 重试重置 | `_reset_for_retry()`         | 设置 `status=RUNNING`，保留重试计数器 |
| 重复重置 | `_reset_for_repeat()`        | 调用 `reset(False)`，保留重复计数器   |

**重要：** `_reset_for_retry()` 和 `_reset_for_repeat()` 必须将 `status` 设置为 `RUNNING`，确保节点在重试/重复期间返回正确状态。

**`_reset_for_retry()` 完整实现：**

```python
def _reset_for_retry(self) -> None:
    """重试时重置状态（保留重试计数器）"""
    self.status = NodeStatus.RUNNING
    self._tick_count = 0
    self._start_time = None
    self._child_index = 0
    self._children_running = False
    
    # CompositeNode 子类字段
    if hasattr(self, 'current_index'):
        self.current_index = 0
    if hasattr(self, '_last_child_finish_time'):
        self._last_child_finish_time = None
    
    # 递归重置所有子节点
    for child in self.children:
        child.reset()
```

**各组合节点的 `_reset_for_retry()` 额外重置字段：**

| 节点类型         | 额外重置字段                                                                                                               |
| ------------ | -------------------------------------------------------------------------------------------------------------------- |
| SequenceNode | `_last_child_finish_time = None`                                                                                     |
| SelectorNode | `_last_child_time = 0`                                                                                               |
| ParallelNode | `cached_statuses.clear()`                                                                                            |
| RandomNode   | `_executed_indices.clear()`, `_success_indices.clear()`, `_failure_indices.clear()`, `_current_running_index = None` |

### 1.2 复合节点 (CompositeNode)

复合节点是行为树的核心控制结构，包含多个子节点并按特定策略执行。

#### 1.2.1 顺序节点 (SequenceNode)

**语义：** 按顺序依次执行子节点，所有子节点成功才返回成功，任一失败立即返回失败。

**支持参数：**

| 参数                    | 说明             | 默认值   |
| --------------------- | -------------- | ----- |
| continue\_on\_failure | 失败后是否继续执行后续子节点 | False |
| childinterval         | 子节点执行间隔（毫秒）    | 0     |
| childinterval\_random | 子节点间隔随机范围（±毫秒） | 0     |

**实现伪代码：**

```
function SequenceNode.tick(context):
    if children is empty:
        return SUCCESS
    
    while current_index < len(children):
        child = children[current_index]
        
        if not child.config.enabled:
            current_index += 1
            continue
        
        status = child.tick(context)
        
        if status == RUNNING:
            return RUNNING          // 等待当前子节点完成
        
        if status == FAILURE:
            if continue_on_failure:
                current_index += 1  // 继续执行下一个子节点
                continue
            current_index = 0       // 重置，下次从头开始
            return FAILURE          // 短路：任一失败即失败
        
        // 子节点执行成功，添加间隔
        if childinterval > 0:
            actual_interval = get_random_duration(childinterval, childinterval_random)
            sleep(actual_interval / 1000.0)
        
        current_index += 1          // 成功则继续下一个
    
    current_index = 0               // 全部成功，重置
    return SUCCESS
```

**关键实现细节：**

- `current_index` 记录当前执行位置，支持 RUNNING 状态的记忆
- 失败时重置 `current_index = 0`，确保下次 tick 从头开始
- 跳过 `enabled = False` 的子节点

#### 1.2.2 选择节点 (SelectorNode)

**语义：** 按顺序执行子节点，任一成功即返回成功，全部失败才返回失败。

**支持参数：**

| 参数                    | 说明             | 默认值 |
| --------------------- | -------------- | --- |
| childinterval         | 子节点执行间隔（毫秒）    | 0   |
| childinterval\_random | 子节点间隔随机范围（±毫秒） | 0   |

**实现伪代码：**

```
function SelectorNode.tick(context):
    if children is empty:
        return FAILURE
    
    while current_index < len(children):
        child = children[current_index]
        
        if not child.config.enabled:
            current_index += 1
            continue
        
        status = child.tick(context)
        
        if status == RUNNING:
            return RUNNING
        
        if status == SUCCESS:
            current_index = 0       // 重置
            return SUCCESS          // 短路：任一成功即成功
        
        if childinterval > 0:
            actual_interval = get_random_duration(childinterval, childinterval_random)
            sleep(actual_interval / 1000.0)
        
        current_index += 1          // 失败则尝试下一个
    
    current_index = 0
    return FAILURE                  // 全部失败
```

#### 1.2.3 并行节点 (ParallelNode)

**语义：** 同时执行所有子节点，根据成功策略决定返回条件。

**成功策略：**

- `require_all`：所有启用的子节点成功才返回成功
- `require_one`：任一子节点成功即返回成功

**实现伪代码：**

```
function ParallelNode.tick(context):
    if children is empty:
        return SUCCESS
    
    success_count = 0
    failure_count = 0
    running_children = []
    
    for i, child in enumerate(children):
        if not child.config.enabled:
            continue
        
        // 检查缓存状态（已完成的子节点不再重复执行）
        if i in cached_statuses:
            status = cached_statuses[i]
            if status == SUCCESS: success_count += 1; continue
            if status == FAILURE: failure_count += 1; continue
        
        status = child.tick(context)
        
        if status == SUCCESS:
            cached_statuses[i] = SUCCESS
            success_count += 1
        elif status == FAILURE:
            cached_statuses[i] = FAILURE
            failure_count += 1
        elif status == RUNNING:
            running_children.append(child)
    
    // require_one 策略：任一成功即返回
    if success_policy == REQUIRE_ONE and success_count > 0:
        abort_running(context, running_children)
        return SUCCESS
    
    if running_children:
        return RUNNING
    
    // 所有子节点执行完毕
    if success_policy == REQUIRE_ALL:
        return SUCCESS if success_count == enabled_count else FAILURE
    else:
        return SUCCESS if success_count > 0 else FAILURE
```

**关键实现细节：**

- `cached_statuses` 缓存已完成子节点的状态，避免重复 tick
- `require_one` 模式下，一旦有子节点成功，立即中止其他运行中的子节点
- `reset()` 时清空缓存

#### 1.2.4 随机节点 (RandomNode)

**语义：** 随机执行子节点，根据成功策略决定返回条件。

**成功策略：**

- `require_all`：所有启用的子节点成功才返回成功
- `require_one`：任一子节点成功即返回成功

**参数配置：**

| 参数                    | 说明                              | 默认值            |
| --------------------- | ------------------------------- | -------------- |
| success\_policy       | 成功策略（require\_all/require\_one） | "require\_all" |
| continue\_on\_failure | 失败后是否继续执行（仅 require\_all 策略生效）  | False          |
| fully\_random         | 完全随机模式（True 时已执行的子节点仍可被选中）      | False          |

**关键实现细节：**

- `_executed_indices`：记录已执行的子节点索引（用于非完全随机模式）
- `_success_indices`：记录成功的子节点索引
- `_failure_indices`：记录失败的子节点索引
- `_current_running_index`：当前正在执行的子节点索引（用于恢复 RUNNING 状态）
- `fully_random=True` 时，子节点可被重复选中，直到所有子节点都执行过一次

#### 1.2.5 开始节点 (StartNode)

**语义：** 作为行为树的根节点，顺序执行所有子节点，子节点失败后继续执行后续子节点。

**与 SequenceNode 的区别：**

| 特性   | SequenceNode      | StartNode         |
| ---- | ----------------- | ----------------- |
| 失败处理 | 任一失败立即返回          | 失败后继续执行           |
| 执行模式 | 短路执行              | 全部执行              |
| 用途   | 条件序列              | 行为树根节点            |

**关键实现细节：**

- `current_index` 记录当前执行位置，支持 RUNNING 状态的记忆
- 子节点失败时设置 `has_failure = true`，但继续执行后续子节点
- **始终返回 SUCCESS**：即使有子节点失败，确保行为树可以继续执行

#### 1.2.6 子树节点 (SubtreeNode)

**语义：** CompositeNode 的子类，在运行时动态加载并执行外部行为树项目。

**参数配置：**

| 参数              | 说明          | 默认值       |
| --------------- | ----------- | ---------- |
| subtree\_path   | 子树项目路径     | ""         |
| blackboard\_mode | 黑板隔离模式     | "inherit"  |
| namespace       | 命名空间前缀     | ""         |
| auto\_reload    | 自动重载       | True       |

**关键设计要点：**

| 要点   | 说明                                                    |
| ---- | ----------------------------------------------------- |
| 延迟加载 | 首次 tick 时才加载子树，避免启动时开销                                |
| 路径解析 | 支持相对路径和绝对路径，相对路径基于 project\_root 解析                   |
| 文件查找 | 按 project.json->main\_tree / tree.json / 单一json 的优先级查找 |
| 循环检测 | 使用 CycleDetector 在加载时 DFS 检测循环引用                      |
| 黑板隔离 | 三种模式：inherit/isolated/namespaced                      |
| 状态传播 | reset() 和 abort() 自动传播到子树所有节点                         |

### 1.3 条件节点 (ConditionNode)

条件节点检测环境状态，返回 SUCCESS 或 FAILURE，不产生副作用。

**基类实现：**

```python
class ConditionNode(Node):
    def __init__(self, node_id=None, config=None):
        super().__init__(node_id, config)
        self.invert = self.config.get("invert", False)           # 结果取反
        self.check_interval_ms = self.config.get("check_interval_ms", 300)  # 检测间隔
        self._last_check_time = 0
        self._children_running = False  # 子节点执行状态追踪标志
        
        # 坐标偏移参数 (向后兼容：支持旧的 offset_x/offset_y 格式)
        offset = self.config.get("offset", None)
        if offset is not None:
            if isinstance(offset, (list, tuple)) and len(offset) >= 2:
                self.offset = (int(offset[0]), int(offset[1]))
            else:
                self.offset = (0, 0)
        else:
            offset_x = self.config.get_int("offset_x", 0)
            offset_y = self.config.get_int("offset_y", 0)
            self.offset = (offset_x, offset_y)
    
    def tick(self, context):
        // 检查子节点是否正在执行
        if self._children_running and self.children:
            return self._execute_children(context)
        
        // 检测间隔控制
        current_time = context.elapsed_time * 1000
        if current_time - self._last_check_time < self.check_interval_ms:
            return self.status    // 未到检测时间，返回上次状态
        
        self._last_check_time = current_time
        result = self._check_condition(context)   // 子类实现
        
        if self.invert:
            result = not result
        
        status = SUCCESS if result else FAILURE
        self.status = status
        
        // 条件成功时执行子节点
        if status == SUCCESS and self.children:
            self._children_running = True
            return self._execute_children(context)
        
        return status
```

**装饰参数（所有条件节点通用）：**

| 参数                  | 说明          | 默认值    |
| ------------------- | ----------- | ------ |
| invert              | 结果取反        | False  |
| retry\_count        | 失败重试次数      | 0      |
| timeout\_ms         | 超时时间        | 0      |
| check\_interval\_ms | 检测间隔        | 300    |
| offset              | 坐标偏移 (x, y) | (0, 0) |
| region\_mode        | 区域选择方式    | "fixed" |

**动态区域检测：**

条件节点支持两种区域选择方式，通过 `region_mode` 配置切换：

- **固定区域检测**（`"fixed"`，默认）：使用 `region` 字段指定的固定坐标区域
- **动态区域检测**（`"dynamic"`）：根据黑板锚点位置 + 偏移量动态计算检测区域

**坐标自动拆分：**

`_save_position` 方法在保存位置到黑板时，自动将坐标拆分为 x/y 分量：

```python
def _save_position(self, context, position):
    if position and hasattr(context, 'blackboard'):
        context.blackboard.set(self.position_key, position)
        context.blackboard.set("last_detection_x", position[0])
        context.blackboard.set("last_detection_y", position[1])
```

#### 1.3.1 文字检测节点 (OCRConditionNode)

**执行流程：**

```
1. context.get_screenshot(region)     → 获取截图
2. context.perform_ocr(image, keywords, language)  → OCR识别
3. 若找到关键词:
   - 收集所有匹配位置，按 search_direction 排序
   - blackboard.set(position_key, position)  → 记录位置
   - return True
4. 若未找到:
   - 输出识别到的所有文本内容
   - return False
```

**参数说明：**

| 参数                | 说明                        | 默认值           |
| ----------------- | ------------------------- | ------------- |
| region            | 检测区域                      | 全屏            |
| keywords          | 检测关键词                     | —             |
| language          | OCR语言（简体中文/English/繁体中文）  | "简体中文"        |
| preprocess\_mode  | 图像预处理模式（默认/复杂色彩/自适应/自动调优） | "默认"          |
| search\_direction | 识别方向（左上/右上/左下/右下）         | "左上"          |
| position\_key     | 位置保存的黑板变量名                | ""（空时使用全局默认值） |
| offset            | 坐标偏移量                     | (0, 0)        |

#### 1.3.2 图像匹配节点 (ImageConditionNode)

**执行流程：**

```
1. Image.open(template_path)          → 加载模板图像（首次缓存）
2. context.get_screenshot(region)     → 获取截图
3. ImageProcessor.find_template(screenshot, template, threshold)  → 模板匹配
   返回: (found: bool, position: Optional[Tuple], confidence: float)
4. 若匹配成功:
   - blackboard.set(position_key, position)  → 记录位置
   - return True
5. 若匹配失败:
   - 输出置信度和阈值信息
   - return False
```

**参数说明：**

| 参数             | 说明                 | 默认值           |
| -------------- | ------------------ | ------------- |
| region         | 检测区域               | 全屏            |
| template\_path | 模板图像路径             | —             |
| threshold      | 匹配阈值（支持百分比和小数两种格式） | 0.8           |
| position\_key  | 位置保存的黑板变量名         | ""（空时使用全局默认值） |

**threshold 参数兼容性：** 百分比格式 `65` 表示 65%，自动转换为 `0.65`；小数格式 `0.65` 直接使用。

#### 1.3.3 颜色检测节点 (ColorConditionNode)

**执行流程：**

```
1. context.get_screenshot(region)     → 获取截图

3. 若找到匹配颜色且匹配像素数 >= min_pixels:

```

**参数说明：**

| 参数            | 说明         | 默认值           |
| ------------- | ---------- | ------------- |
| region        | 检测区域       | 全屏            |
| target\_color | 目标颜色       | —             |
| tolerance     | 颜色容差       | 30            |
| min\_pixels   | 最小匹配像素数    | 1             |
| position\_key | 位置保存的黑板变量名 | ""（空时使用全局默认值） |

#### 1.3.4 数字比较节点 (NumberConditionNode)

**执行流程：**

```
1. context.get_screenshot(region)     → 获取截图
2. OCRManager.recognize_number_with_position(...)  → 数字OCR
3. 若识别成功:
   - 收集所有数字位置，按 search_direction 排序
   - blackboard.set("last_number_value", value)
   - blackboard.set(position_key, position)
   - blackboard.set(value_key, value)

4. 若识别失败: return False
```

| 参数                | 说明                      | 默认值                   |
| ----------------- | ----------------------- | --------------------- |
| region            | 检测区域                    | 全屏                    |
| preprocess\_mode  | 图像预处理（默认/复杂色彩/自适应/自动调优） | "默认"                  |
| extract\_mode     | 提取模式（无规则/x/y/自定义）       | "无规则"                 |
| extract\_pattern  | 自定义提取模式（使用\*作为通配符）      | ""                    |
| compare\_mode     | 比较运算符（>、<、>=、<=、==、!=）  | ">"                   |
| threshold         | 比较目标值                   | 0                     |
| min\_confidence   | 最小置信度                   | 0.5                   |
| search\_direction | 识别方向（左上/右上/左下/右下）       | "左上"                  |
| value\_key        | 数字值保存的黑板变量名             | "last\_number\_value" |
| position\_key     | 检测位置保存的黑板变量名            | ""（空时使用全局默认值）         |
| offset            | 坐标偏移量（相对偏移）             | (0, 0)                |

#### 1.3.5 变量判断节点 (VariableConditionNode)

**语义：** 判断黑板变量是否满足指定条件，支持常量值比较、变量间比较和存在性判断。

**支持参数：**

| 参数 | 说明 | 默认值 |
|------|------|--------|
| variable_name | 要判断的黑板变量名 | "" |
| operator | 比较运算符（==, !=, >, >=, <, <=, exists, not_exists） | "==" |
| compare_type | 比较值类型（常量值/变量名） | "constant" |
| compare_value | 比较值（常量值模式时显示） | "" |
| compare_variable | 比较变量（变量名模式时显示） | "" |

**关键实现细节：**

- `compare_type == "variable"` 时，比较值从黑板动态获取，保留原始类型
- 比较变量不存在或值为 None 时，显式返回 False 并输出日志
- `exists/not_exists` 运算符仅调用 `blackboard.exists()`，不获取变量值

#### 1.3.6 文本提取节点 (TextExtractNode)

**语义：** 通过 OCR 识别屏幕指定区域的文字内容，提取文本并保存到黑板变量。

**支持参数：**

| 参数               | 说明                   | 默认值                   |
| ---------------- | -------------------- | --------------------- |
| region           | 检测区域                 | None                  |
| extract\_mode    | 提取模式（all/keywords）   | all                   |
| keywords         | 关键词（keywords模式时使用）   | ""                    |
| language         | OCR识别语言              | chi\_sim              |
| preprocess\_mode | 图像预处理模式（normal/game） | normal                |
| output\_key      | 提取文本输出变量名            | last\_extracted\_text |
| save\_all\_text  | 是否保存全部OCR文本          | False                 |
| all\_text\_key   | 全部文本变量名              | all\_ocr\_text        |
| save\_position   | 是否保存检测位置             | True                  |
| position\_key    | 位置变量名                | ""（空时使用全局默认值）         |
| offset           | 坐标偏移                 | (0, 0)                |

### 1.4 动作节点 (ActionNode)

动作节点执行具体操作，产生副作用。

**基类实现：**

```python
class ActionNode(Node):
    def __init__(self, node_id=None, config=None):
        super().__init__(node_id, config)
        self._children_running = False  # 子节点执行状态追踪标志
    
    def tick(self, context):
        if self._children_running and self.children:
            return self._execute_children(context)
        
        status = self._execute_action(context)   // 子类实现
        self.status = status
        
        if status == SUCCESS and self.children:
            context.notify_node_status(self.node_id, "success")
            self._children_running = True
            return self._execute_children(context)
        
        return status
```

**装饰参数（所有动作节点通用）：**

| 参数            | 说明   | 默认值 |
| ------------- | ---- | --- |
| repeat\_count | 重复次数 | 0   |
| timeout\_ms   | 超时时间 | 0   |

**随机范围参数（部分动作节点支持）：**

| 节点             | 参数              | 随机范围参数                  |
| -------------- | --------------- | ----------------------- |
| DelayNode      | duration\_ms    | duration\_ms\_random    |
| MouseClickNode | click\_interval | click\_interval\_random |
| MouseMoveNode  | move\_duration  | move\_duration\_random  |
| KeyPressNode   | duration        | duration\_random        |

**随机值计算函数：**

```python
def get_random_value(base_value: int, random_range: int = 0, min_value: int = None) -> int:
    if random_range <= 0:
        return base_value
    min_val = base_value - random_range
    max_val = base_value + random_range
    if min_value is not None:
        min_val = max(min_val, min_value)
    return random.randint(int(min_val), int(max_val))
```

## 5 输入系统实现

### 5.1 多引擎输入架构

系统支持 4 种输入引擎，键盘和鼠标引擎可独立选择：

| 引擎               | 类型   | InputLevel   | 说明                     |
| ---------------- | ---- | ------------ | ---------------------- |
| PyAutoGUI         | 键盘+鼠标 | APPLICATION  | 标准 Python 库，应用层输入       |
| DD                | 键盘+鼠标 | DRIVER       | DD虚拟键盘驱动，驱动层输入         |
| IbInputSimulator  | 键盘+鼠标 | DRIVER       | IB驱动模拟，支持多种驱动子模式       |
| Background        | 键盘+鼠标 | BACKGROUND   | PostMessage 后台输入，需绑定窗口  |

**配置方式：**

```yaml
input:
  keyboard_method: "pyautogui"   # pyautogui | dd | ib | background
  mouse_method: "pyautogui"      # pyautogui | dd | ib | background
```

**架构层次：**

```
ExecutionContext
    ├── InputControllerManager.get_keyboard_engine()
    │     ├── PyAutoGUIKeyboardController
    │     ├── DDKeyboardController
    │     ├── IBKeyboardController
    │     └── BackgroundKeyboardController
    └── InputControllerManager.get_mouse_engine()
          ├── PyAutoGUIMouseController
          ├── DDMouseController
          ├── IBMouseController
          └── BackgroundMouseController
```

### 5.2 IbInputSimulator 关键 API

**DLL API：**

```
IbSendInit(send_type, mode, args)    // 初始化，send_type 选择驱动子模式
IbSendInputHook(mode)                // Hook 开关：1=ON, 0=OFF
IbSendDestroy()                      // 销毁，释放资源
```

**send_type 映射：**

| send_type | 驱动模式       | 说明                   |
| --------- | ---------- | -------------------- |
| 0         | AnyDriver  | 自动选择可用驱动             |
| 1         | SendInput  | Win32 SendInput，移除注入标志 |
| 2         | Logitech   | 罗技驱动模拟               |
| 3         | Razer      | 雷蛇驱动模拟               |
| 5         | MCII       | 内核级鼠标注入，需目标 PID      |
| 6         | GHubNew    | 罗技 G Hub 新版驱动        |

**Hook 机制：** 每次输入操作：`IbSendInputHook(1)` → `SendInput(...)` → `IbSendInputHook(0)`。Hook 拦截 SendInput 调用，移除 `injected` 标志。

**ctypes 调用约定：** IB DLL 使用 `__cdecl` 调用约定，加载方式为 `ctypes.cdll.LoadLibrary(dll_path)`。

### 5.3 后台输入实现

BackgroundInputController 基于 Win32 PostMessage 实现，无需前台窗口焦点。坐标使用客户区坐标（非屏幕坐标），通过 `ScreenToClient(hwnd, point)` 转换。

### 5.4 DD虚拟键盘实现

**DLL 加载搜索路径优先级：** 构造参数 → `bt_utils/DD64.dll` → `bt_utils/drivers/DD64.dll` → `../drivers/DD64.dll` → 项目根/drivers/DD64.dll

### 5.5 InputControllerFactory

输入控制器工厂负责根据配置创建对应的输入引擎实例，文件路径为 `bt_utils/input_controller_factory.py`。

## 6 画布编辑器实现

### 6.1 坐标系统

画布使用三层坐标变换：

```
screen_x = world_x * zoom + pan_x
screen_y = world_y * zoom + pan_y

world_x = (screen_x - pan_x) / zoom
world_y = (screen_y - pan_y) / zoom
```

### 6.2 节点渲染

NodeItem 在 Tkinter Canvas 上绘制，包含阴影、主体矩形、左侧分类色条、节点名称、状态指示器、输入/输出端口等视觉元素。

### 6.3 连线渲染

连线使用贝塞尔曲线风格的三段折线（垂直段→水平段→垂直段）。

### 6.3.1 Canvas 虚拟化

为支持数百节点流畅交互，画布新增虚拟化渲染机制，仅渲染视口内可见节点。拖拽节点时仅重绘关联连线，使用 `_node_connections_map` 反向索引。

### 6.4 连接验证机制

画布在创建连接时进行严格验证：

1. 检查节点是否存在
2. 检查是否已经存在相同的连接
3. 检查子节点是否已经有父节点
4. 检查是否会形成循环（DFS 算法）

连接失败时通过日志管理器输出提示信息，不使用弹窗干扰用户操作。

### 6.5 自动布局

当加载的树数据不包含位置信息时，使用递归自动布局，基于子树宽度计算节点位置。

## 8 OCR 管理器实现

### 8.1 RapidOCR 引擎初始化

OCRManager 封装 RapidOCR + ONNX Runtime，无需安装 Tesseract：

```python
@singleton
class OCRManager:
    def __init__(self):
        self._engine = RapidOCR(params={
            "Det.limit_side_len": 736,
            "Det.limit_type": "max"  # 只缩小大图，不放大小图
        })
        self._auto_config_selector = AutoConfigSelector()
```

### 8.2 图像预处理模式

| 模式             | 处理流程                    | 适用场景      |
| -------------- | ----------------------- | --------- |
| normal (默认)    | 灰度→对比度增强→锐化→二值化         | 标准文字识别    |
| game (复杂色彩)    | 放大→中值滤波→灰度→对比度增强→锐化→二值化 | 游戏界面、小字体  |
| adaptive (自适应) | CLAHE对比度增强→自适应二值化       | 低对比度、渐变背景 |
| auto (自动调优)    | 根据图像特征自动选择最佳预处理参数       | 所有场景，推荐使用 |

### 8.3 结果缓存机制

OCRManager 内置线程安全的结果缓存：

- **TTL 过期**：缓存超过 1 秒自动失效
- **LRU 淘汰**：缓存条目超过 200 条时淘汰最旧的
- **线程安全**：使用 `threading.Lock` 保护缓存操作
- 缓存键基于图像哈希和参数哈希计算

### 8.4 图像特征分析器

`ImageFeatureAnalyzer` 自动分析图像特征（亮度、对比度、噪点水平、字符高度、背景类型等），为预处理决策提供依据。

### 8.5 智能放大策略

`SmartScaleStrategy` 基于字符高度计算安全的放大倍数（最大 4 倍），使用 Lanczos 高质量插值放大图像。

### 8.6 自动配置选择器

`AutoConfigSelector` 根据图像特征自动选择最佳预处理参数，按优先级处理：噪点 > 低对比度 > 深色背景 > 渐变背景 > 小字体。

### 8.7 精确关键词定位

基于字符实际宽度系数（中文 2.0、大写 1.2、小写 1.0、数字 1.0 等）精确计算关键词在文本行中的位置，解决中英混合文本定位偏差问题。

## 14 UI 更新分发器实现（防止假卡死）

### 14.1 问题背景

tkinter 的 GUI 操作必须在主线程执行，但行为树引擎在独立线程中运行。直接从后台线程调用 tkinter 可能导致界面假卡死。

### 14.2 核心解决方案

采用**定时轮询 + 后台守护**混合模式：

- 运行时 10ms 轮询间隔，空闲时 100ms 轮询间隔
- 轮询必须在编辑器初始化时启动，而不是运行时启动
- **不要停止轮询**，让它持续运行
- 当 `_widget` 为 None 时，使用后台线程守护，等待 `_widget` 可用

### 14.3 防止假卡死的关键要点

| 问题                  | 错误做法                       | 正确做法                            |
| ------------------- | -------------------------- | ------------------------------- |
| 启动时假卡死              | 在 `_start_running()` 中启动轮询 | 在 `_init_ui_dispatcher()` 中启动轮询 |
| 手动停止后异常             | 停止时调用 `stop_polling()`     | **不要停止轮询**，让它持续运行               |
| 轮询永久停止              | `_widget` 为 None 时直接返回     | 使用后台线程守护，等待 `_widget` 可用        |

## 20 窗口绑定功能实现

### 20.1 WindowManager 窗口管理器

WindowManager 使用 ctypes 调用 Windows API，提供窗口枚举、查找、切换等功能。

**核心 API：**

| 方法 | 说明 |
|------|------|
| `enum_all_windows()` | 枚举所有可见窗口 |
| `find_window_by_title(keyword)` | 根据标题关键字查找窗口 |
| `find_window_by_pid(pid)` | 通过进程ID查找窗口 |
| `find_main_window_by_pid(pid)` | 智能查找主窗口（按优先级排序） |
| `find_window_smart(pid, title)` | 智能查找：优先PID，其次关键字 |
| `find_window_smart_with_hwnd(hwnd, pid, title)` | 三级降级：句柄>PID>标题 |
| `switch_to_window(hwnd)` | 切换到指定窗口（置顶并激活） |
| `restore_window(hwnd)` | 恢复窗口（取消置顶） |
| `get_window_rect(hwnd)` | 获取窗口矩形区域 |
| `get_window_pid(hwnd)` | 获取窗口的进程ID |

### 20.2 ExecutionContext 窗口绑定扩展

ExecutionContext 扩展了窗口绑定功能：

- `bind_window(hwnd)` / `get_bound_window()`：绑定/获取目标窗口
- `get_screenshot(region)`：支持后台截图，绑定窗口时使用 `WindowCapture`
- `convert_to_screen_coords()` / `convert_to_window_coords()`：坐标转换
- `smart_switch_to_bound_window()`：智能切换到绑定窗口（保存原前台窗口）
- `smart_restore_foreground_window()`：智能恢复前台窗口
- 帧级截图缓存：同一 tick 内相同 region 只截图一次

### 20.3 StartNode 窗口绑定实现

StartNode 在执行时绑定窗口，支持句柄>PID智能>标题三级降级查找。找到窗口后自动更新 `window_hwnd` 配置。

### 20.4 ActionNode 智能窗口切换

ActionNode 在执行动作前自动切换窗口，执行后恢复原前台窗口。通过 `SKIP_WINDOW_SWITCH` 类属性跳过不需要窗口操作的节点（DelayNode、SetVariableNode、AlarmNode、CodeNode、ScriptNode、StartTreeNode、StopTreeNode）。

### 20.5 WindowSelectField GUI 组件

提供窗口选择下拉框，选择窗口时自动保存句柄(hwnd)和PID。同标题多窗口时显示 PID 后缀区分。

### 20.6 RegionField 坐标转换集成

RegionField 在选择区域时自动检测绑定窗口，将屏幕坐标转换为窗口相对坐标。

### 20.7 执行流程

```
1. StartNode 初始化 → 读取窗口配置 → 查找窗口 → context.bind_window(hwnd)
2. 条件节点执行 → context.get_screenshot() → WindowCapture 后台截图
3. 动作节点执行 → smart_switch → 执行动作 → smart_restore
```

## 21 子树引用与多行为树并行实现

### 21.1 CycleDetector 循环检测

使用 DFS 算法检测子树引用中的循环依赖，时间复杂度 O(V + E)。

### 21.2 NamespacedBlackboard 命名空间黑板

通过键名前缀实现数据隔离，所有读写操作代理到父级 Blackboard：

| 模式         | 数据可见性 |
| ---------- | ----- |
| inherit    | 完全共享  |
| isolated   | 完全隔离  |
| namespaced | 前缀隔离  |

### 21.3 MultiTreeManager 多树管理器

管理多个独立的行为树实例，支持线程安全的并发操作（add/remove/start/stop）。

### 21.4 GuiTabManager 多 Tab 管理器

继承 MultiTreeManager，增加 Tab 切换和 UI 回调支持。顶部"开始"/"停止"按钮控制所有 Tab，Tab 内按钮控制单个。

### 21.5 代理属性模式

BehaviorTreeEditor 使用代理属性模式，将 `self.canvas/engine/context` 等属性代理到当前活动 Tab。

### 21.6 Tab 持久化

关闭应用时保存 Tab 列表到 SettingsManager，启动时恢复。

### 21.7 LogManager 按 Tab 隔离

LogManager 支持按 Tab 名管理停止状态，停止单个 Tab 不影响其他 Tab 日志。

### 21.8 SubtreeFileWatcher 文件监控

通过比较文件修改时间(mtime)检测子树文件变化，支持热重载场景。

### 21.9 SubtreeNodeItem GUI 实现

SubtreeNodeItem 是 NodeItem 的子类，提供子树节点的可视化渲染和预览交互（虚线边框、紫色色条、展开/折叠预览缩略图）。

### 21.10 FolderField 文件夹选择字段

选择子树项目文件夹后，自动复制到项目的 `subtrees/` 目录，旧目录移到 cache 而非直接删除。
