## 更新日志

**V1.6.0**

**新增内容**

- **新增启动/停止树节点**：
  - 启动树节点，用于在行为树运行时启动其他已加载的行为树
  - 停止树节点，用于在行为树运行时停止其他行为树或当前行为树
- **新增单树快捷键功能**：
  - 在设置界面为每个行为树绑定独立的启动/停止快捷键，同一快捷键切换运行状态
- **键鼠输入分离**：键盘和鼠标的输入引擎现在可以独立选择，支持自由组合
  - 现在可支持的引擎选项：PyAutoGUI、DD虚拟键盘、IbInputSimulator、后台消息
  - 支持混合模式，如键盘用IB驱动层+鼠标用PyAutoGUI，或键盘用后台消息+鼠标用PyAutoGUI
- **IbInputSimulator 多驱动输入引擎**：新增基于 IbInputSimulator.dll 的多驱动输入引擎
  - 支持驱动模式：自动检测、Logitech LGS、Logitech G HUB (新版)、Razer Synapse、MouClassInputInjection、SendInput
  - SendInput 模式：去除 injected 标志，伪装为硬件输入，能绕过部分反作弊检测
  - MouClassInputInjection 模式：内核级鼠标注入，反检测能力最强（需指定目标进程PID）
  - 需要管理员权限运行
- **坐标浮动**：鼠标点击和鼠标移动节点新增 X/Y 坐标浮动参数
  - 每次点击/移动时，坐标会在基础值 ± 浮动范围内随机偏移
- **前端画布性能优化**：画布仅渲染视口内可见节点，并修改了切换节点、重绘连线时的逻辑，200+节点场景下拖拽帧率显著提升
- **条件节点运行性能优化：**
  - **帧级截图缓存**：同一tick内相同区域只截图一次，减少并行多个检测时的性能开销
  - **自适应UI轮询间隔**：运行时10ms/空闲时100ms，降低空闲时的CPU开销
  - **OCR缓存容量提升**：从50条提升至200条，新增命中率统计
  - **条件节点智能跳过**：连续5次相同结果时间隔×2，连续15次时间隔×4，减少无效检测
  - **差异截图优化**：未配置检测区域时自动使用上次检测位置附近300×300区域，避免全屏截图

**优化内容**

- **全局快捷键支持组合键绑定**：现在全局快捷键和单树快捷键均支持组合键（如 Shift+F10、Ctrl+Alt+F1 等）

* **后台消息引擎改进**：后台模式下鼠标操作不再强制切换窗口到前台
* **颜色检测采集优化**：采集颜色时增加了放大镜功能

- **预览检测时现在会自动展开运行日志**：防止用户在未展开运行日志时进行预览检测，无法看到检测结果

## 1. 产品概述

### 1.1 什么是行为树？

行为树是一种可视化的自动化流程设计工具，通过图形化界面拖拽节点并连接它们，可以创建复杂的自动化任务。行为树广泛应用于客户端自动化、RPA自动化、测试自动化等领域。

### 1.2 产品定位

AutoDoor 行为树系统是一个独立的可视化行为树编辑与执行框架，面向 Windows 平台的自动化场景

### 1.3 核心特性

| 特性       | 说明                                     |
| -------- | -------------------------------------- |
| 可视化编辑器   | 基于 CustomTkinter 的节点式编辑器，支持拖拽、连线、缩放、框选 |
| 行为树引擎    | 独立线程执行，支持启动/暂停/停止/恢复                   |
| 24 种内置节点 | 5 种组合节点 + 6 种条件节点 + 12 种动作节点 + 1 种开始节点 |
| 黑板系统     | 观察者模式的数据共享机制，节点间解耦通信                   |
| 撤销/重做    | Command 模式，支持 100 步历史                  |
| DD 虚拟键盘  | 硬件模拟输入，绕过大多数输入检测                       |
| OCR 集成   | 内嵌 RapidOCR，支持中英文识别                    |
| 脚本录制     | 录制你的鼠标键盘操作，生成脚本文件                      |
| 项目管理     | 完整的项目文件管理系统，支持导入导出                     |

### 1.4 输入引擎说明

系统提供统一的客户端，支持在设置中灵活选择输入引擎：

| 输入引擎             | 说明                    |
| ---------------- | --------------------- |
| PyAutoGUI        | 标准输入模拟，兼容性好           |
| DD虚拟键盘           | 硬件级模拟输入，绕过大多数输入检测     |
| IbInputSimulator | 驱动层输入，反检测能力最强（需管理员权限） |
| 后台消息             | 通过窗口消息发送输入，不切换前台窗口    |

键盘和鼠标引擎可独立选择，支持混合模式。

- 点击选中节点
- 在右侧属性面板配置参数

**步骤5：保存运行**

- 点击【运行】按钮执行行为树

## 4. 节点类型详解

### 4.1 开始节点

**用途**：行为树的根节点，作为整个行为树的入口点，控制整体执行流程。继承自组合节点，具有组合节点的所有特性。

**特性**：

| 特性   | 说明                   |
| ---- | -------------------- |
| 自动创建 | 新建项目时自动创建，无需手动添加     |
| 不可删除 | 开始节点受保护，无法被删除、复制或剪切  |
| 失败继续 | 子节点失败后继续执行后续子节点（不短路） |
| 循环控制 | 支持重复执行次数控制           |

**执行流程**：

```
        开始
          │
   ┌─────┼─────┐
   ↓     ↓     ↓
子节点1 子节点2 子节点3 ...
   ↓     ↓     ↓
   └─────┼─────┘
         ↓
    重复/完成
```

**参数**：

| 参数     | 说明                            |
| ------ | ----------------------------- |
| 失败重试次数 | 子节点失败后的重试次数（-1 表示无限重试，默认 0）   |
| 重复次数   | 整个行为树的重复执行次数（-1 表示无限循环，默认 -1） |
| 重复间隔   | 重复执行之间的间隔时间（毫秒，默认 100）        |
| 超时时间   | 超时时间（毫秒，0 表示不限时）              |
| 绑定窗口   | 是否启用窗口绑定功能（默认关闭）              |
| 窗口标题   | 目标窗口标题关键字（启用绑定窗口后显示）          |
| 窗口PID  | 目标窗口的进程ID（启用绑定窗口后自动保存）        |
| 窗口句柄   | 目标窗口的句柄hwnd（启用绑定窗口后自动保存）      |

**窗口绑定功能**：

启用窗口绑定后，行为树将绑定到指定窗口，实现以下功能：

| 功能     | 说明                                 |
| ------ | ---------------------------------- |
| 后台截图   | 使用 PrintWindow API 截取后台窗口图像，无需窗口置顶 |
| 自动坐标转换 | 屏幕绝对坐标与窗口相对坐标自动转换                  |
| 智能窗口切换 | 执行动作前自动切换到绑定窗口，执行后恢复原窗口            |

**窗口绑定使用步骤**：

1. 点击开始节点，打开属性面板
2. 勾选"绑定窗口"选项
3. 在"窗口标题"下拉框中选择目标窗口
   - 下拉列表显示所有可见窗口的标题
   - 同标题多窗口时显示"标题 (PID:xxxx)"区分不同进程
   - 支持关键字模糊匹配
   - 点击下拉框自动刷新窗口列表
   - 点击"清空"按钮可取消窗口绑定
4. 配置其他参数后运行行为树

**窗口绑定注意事项**：

- 目标窗口必须处于可见状态（未被最小化）
- 区域选择时会自动转换为窗口相对坐标
- 执行动作时会自动切换到绑定窗口，执行完成后恢复原窗口
- 如果目标窗口未找到，行为树会继续执行但不会绑定窗口
- 延时、设变量、报警、代码、脚本、启动树、停止树节点不会触发窗口切换

**重复次数说明**：

| 次数    | 说明                      |
| ----- | ----------------------- |
| -1    | 无限循环（默认），行为树会一直执行直到手动停止 |
| 0     | 不重复，执行一次后结束             |
| N > 0 | 执行 N 次后结束               |

**使用场景**：

- 作为行为树的入口点
- 控制整体执行流程和重复次数
- 实现持续运行的自动化任务

**示例**：

```
开始节点（重复次数=-1）
├── OCR检测：检测目标
├── 鼠标点击：点击目标
└── 延时：等待1秒
```

**注意事项**：

- 每个行为树有且仅有一个开始节点
- 开始节点会自动创建在画布中央偏上的位置
- 无法通过节点面板手动添加开始节点

### 4.2 组合节点

组合节点是行为树的核心控制结构，包含多个子节点并按特定策略执行。

#### 4.2.1 顺序节点

**用途**：按顺序依次执行子节点，所有子节点成功才返回成功，任一失败立即返回失败。

**执行流程**：

```
          顺序
           │
   ┌───────┼────────┐
   ↓       ↓        ↓
子节点1 → 子节点2 → 子节点3 → ...
  ↓失败     ↓失败     ↓失败
返回失败   返回失败   返回失败
```

**参数**：

| 参数      | 说明                  |
| ------- | ------------------- |
| 子节点间隔   | 每个子节点执行完成后的等待时间（毫秒） |
| 失败后继续执行 | 开启后，中间节点失败仍继续执行后续节点 |

**使用场景**：

- 需要按固定顺序执行的任务
- 流程化的自动化操作
- 登录流程、表单填写等

**示例**：

```
顺序节点
├── OCR检测：检测登录按钮
├── 鼠标点击：点击登录按钮
├── 延时：等待2秒
└── OCR检测：检测登录成功
```

#### 4.2.2 选择节点

**用途**：按顺序执行子节点，任一成功即返回成功，全部失败才返回失败。

**执行流程**：

```

子节点1 → 子节点2 → 子节点3 → ...
  ↓成功     ↓成功     ↓成功
返回成功   返回成功   返回成功
```

**参数**：

| 参数    | 说明                  |
| ----- | ------------------- |
| 子节点间隔 | 每个子节点执行完成后的等待时间（毫秒） |

**使用场景**：

- 多种方案尝试，任一成功即可
- 容错处理，提供备选方案

**示例**：

```
选择节点
├── OCR检测：检测账号密码登录入口
├── OCR检测：检测扫码登录入口
└── OCR检测：检测第三方登录入口
```

#### 4.2.3 并行节点

**用途**：同时执行所有子节点，根据成功策略决定返回条件。

**执行流程**：

```
       并行
         │
   ┌─────┼─────┐
   ↓     ↓     ↓
子节点1 子节点2 子节点3
   ↓     ↓     ↓
   └─────┼─────┘
         ↓
     汇总结果
```

**参数**：

| 参数   | 说明                              |
| ---- | ------------------------------- |
| 成功策略 | 全部成功：所有子节点成功才成功；任一成功：任一子节点成功即成功 |

**使用场景**：

- 同时监控多个条件
- 并行执行独立任务

#### 4.2.4 随机节点

**用途**：随机执行子节点，根据成功策略决定返回条件。

**执行流程**：

```
         随机
           │
   ┌───────┼───────┐
   ↓       ↓       ↓
子节点?  子节点?  子节点?
   │       │       │
   └───────┼───────┘
           ↓
       汇总结果
```

**参数**：

| 参数      | 说明                               |
| ------- | -------------------------------- |
| 成功策略    | 全部成功：所有子节点成功才成功；任一成功：任一子节点成功即成功  |
| 失败后继续执行 | 开启后，子节点失败不会中断随机执行（仅"全部成功"策略生效）   |
| 完全随机    | 开启后，已执行的子节点仍可被再次选中，直到所有子节点都执行过一次 |

**执行逻辑说明**：

| 设置组合             | 执行行为                    |
| ---------------- | ----------------------- |
| 全部成功 + 失败后继续执行关闭 | 遇到失败立即返回失败              |
| 全部成功 + 失败后继续执行开启 | 遇到失败继续执行，所有子节点执行完毕后判断   |
| 任一成功             | 遇到成功立即返回成功              |
| 完全随机关闭           | 每个子节点只执行一次，执行后从随机池移除    |
| 完全随机开启           | 子节点可被重复选中，直到所有子节点都执行过一次 |

**使用场景**：

- 随机选择不同的执行路径
- 模拟随机行为
- 多种方案随机尝试

**示例**：

```
随机节点（成功策略=任一成功）
├── OCR检测：检测入口A
├── OCR检测：检测入口B
└── OCR检测：检测入口C

// 随机选择一个入口检测，成功即返回
```

**使用场景**：

## 6. 连接操作

### 6.1 节点端口说明

每个节点有两个端口：

- **输入端口**：节点顶部的圆点，用于接收父节点的连接
- **输出端口**：节点底部的圆点，用于连接子节点

```
        ○ 输入端口
    ┌─────────┐
    │  节点    │
    └─────────┘
        ○ 输出端口
```

### 6.2 创建连接

1. 将鼠标移到父节点的**输出端口**（底部圆点）
2. 光标变为十字形状时，按住左键拖拽
3. 拖拽到子节点的**输入端口**（顶部圆点）
4. 释放鼠标完成连接

### 6.3 删除连接

| 方法  | 操作              |
| --- | --------------- |
| 快捷键 | 选中连线后按 `Delete` |

### 6.4 连接规则

- 一个输出端口可以连接多个子节点
- 一个输入端口只能连接一个父节点
- 不能创建循环连接（子节点连接回父节点）

### 6.5 执行顺序

当组合节点有多个子节点时，连接线上会显示序号（1、2、3...）表示执行顺序。

## 8. 属性配置

### 8.1 基本属性

所有节点都有的基本属性：

| 属性 | 说明               |
| -- | ---------------- |
| 名称 | 节点的显示名称，便于识别     |
| 启用 | 是否启用该节点（禁用后跳过执行） |

### 8.2 区域选择

条件节点通常需要设置检测区域，支持两种区域选择方式：

#### 8.2.1 固定区域检测（默认）

使用固定坐标区域进行检测，与改造前行为一致。

**手动输入**：

- 在区域输入框输入坐标：`x1,y1,x2,y2`
- 例如：`100,100,300,200`

**使用选择工具**：

1. 点击区域输入框右侧的【选择】按钮
2. 应用窗口自动最小化
3. **放大镜自动显示**，帮助精确定位：
   - 显示4倍放大的屏幕内容
   - 实时显示鼠标坐标
   - 实时显示当前位置颜色值
4. 在屏幕上拖拽选择区域
5. 释放鼠标后自动填入坐标

#### 8.2.2 动态区域检测

根据黑板锚点位置 + 偏移量动态计算检测区域，使检测区域能跟随移动目标。

**切换方式**：在条件节点属性面板顶部，将"区域选择方式"从"固定区域检测"切换为"动态区域检测"。

**参数说明**：

| 参数      | 说明                                   | 默认值                                |
| ------- | ------------------------------------ | ---------------------------------- |
| 设置最近检测点 | 勾选后使用 `last_detection_position` 作为锚点 | ✅ 勾选                               |
| 位置变量名   | 取消勾选"设置最近检测点"后，可输入自定义黑板变量名作为锚点       | 空（空时回退到 `last_detection_position`） |
| 区域偏移量   | 相对于锚点的偏移量 `[x1, y1, x2, y2]`         | `[-50, -50, 50, 50]`               |

**动态区域计算方式**：

```
实际区域 = (锚点x + 偏移x1, 锚点y + 偏移y1, 锚点x + 偏移x2, 锚点y + 偏移y2)
```

例如：锚点为 (400, 300)，偏移量为 \[-20, 30, 80, 60]，则实际区域为 (380, 330, 480, 360)。

**区域偏移量手动输入**：

- 在偏移量输入框输入4个值：`x1, y1, x2, y2`
- 例如：`-20, -10, 80, 10`

**区域偏移量测量**：

1. 点击偏移量输入框右侧的【测量】按钮
2. 应用窗口自动最小化
3. **第一步**：点击屏幕上的锚点（参考位置，如某个图标中心）
4. **第二步**：按住鼠标拖拽选择区域范围，实时显示矩形边框
5. 释放鼠标后自动计算偏移量并填入

**典型使用场景**：

1. 先用图像匹配节点（固定区域）检测角色头像位置
2. 再用文字检测节点（动态区域检测）以头像为锚点，偏移量设为头像下方区域
3. 头像移动后，检测区域自动跟随

### 8.3 放大镜功能

在区域选择、截图选择、位置选择时，系统会自动显示放大镜窗口，帮助精确定位：

**放大镜特性**：

| 特性    | 说明          |

### 8.6 坐标偏移测量

条件节点（OCR检测、图像匹配、颜色检测、数字比较）支持坐标偏移功能，可以将检测到的位置偏移后再保存到黑板：

**使用偏移测量工具**：

1. 在条件节点的属性面板中找到【坐标偏移】字段
2. 点击【测量】按钮
3. **第一步**：点击参考点（检测到的原始位置）
4. **第二步**：点击目标点（实际需要点击的位置）
5. 偏移值自动计算并填入

**偏移测量流程**：

```
┌─────────────────────────────────────────────────────────────┐
│                     偏移测量流程                              │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  1. 点击【测量】按钮                                         │
│       ↓                                                     │
│  2. 放大镜显示，点击参考点（如按钮中心）                      │
│       ↓                                                     │
│  3. 放大镜继续显示，点击目标点（如按钮边缘）                  │
│       ↓                                                     │
│  4. 自动计算偏移值：(目标X-参考X, 目标Y-参考Y)               │
│       ↓                                                     │
│  5. 偏移值填入输入框                                         │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

**使用场景**：

- 检测到按钮位置后，需要点击按钮的特定边缘
- OCR识别文字后，需要点击文字旁边的图标
- 图像匹配后，需要点击匹配图像的相对位置

**区域偏移量测量**（动态区域检测模式下）：

动态区域检测模式下的"区域偏移量"字段也支持测量，但测量方式为两步操作：

1. 在条件节点的属性面板中，将"区域选择方式"切换为"动态区域检测"
2. 在【区域偏移量】字段点击【测量】按钮
3. **第一步**：点击锚点（参考位置，如某个图标中心）
4. **第二步**：按住鼠标拖拽选择区域范围，实时显示矩形边框
5. 释放鼠标后自动计算偏移量并填入

```
┌─────────────────────────────────────────────────────────────┐
│                  区域偏移量测量流程                            │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  1. 点击【测量】按钮                                         │
│       ↓                                                     │
│  2. 放大镜显示，点击锚点（参考位置，如图标中心）              │
│       ↓                                                     │
│  3. 按住鼠标拖拽选择区域范围（实时显示矩形边框）              │
│       ↓                                                     │
│  4. 释放鼠标，自动计算偏移量：                                │
│     x1 = min(拖拽起点X, 拖拽终点X) - 锚点X                  │
│     y1 = min(拖拽起点Y, 拖拽终点Y) - 锚点Y                  │
│     x2 = max(拖拽起点X, 拖拽终点X) - 锚点X                  │
│     y2 = max(拖拽起点Y, 拖拽终点Y) - 锚点Y                  │
│       ↓                                                     │
│  5. 偏移量填入输入框，格式：x1, y1, x2, y2                  │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 8.7 随机范围设置

为了模拟更自然的人类操作，部分节点支持为时长参数设置随机范围：

**支持的节点和参数**：

| 节点     | 参数   | 随机范围说明            |
| ------ | ---- | ----------------- |
| 延时节点   | 延时时长 | 实际延时 = 基础值 ± 随机范围 |
| 鼠标点击节点 | 点击间隔 | 实际间隔 = 基础值 ± 随机范围 |
| 鼠标移动节点 | 移动时长 | 实际时长 = 基础值 ± 随机范围 |

**设置随机范围**：

1. 在节点属性面板中找到时长参数
2. 在参数右侧找到【随机范围】输入框
3. 输入随机范围值（毫秒）
4. 实际执行时会在 \[基础值-范围, 基础值+范围] 之间随机选择

**示例**：

```
延时时长: 1000ms
随机范围: 200ms

实际延时范围: 800ms ~ 1200ms
每次执行随机选择一个值
```

**使用建议**：

- 随机范围建议设置为时长的 10%\~20%
- 适当的随机变化可以模拟更自然的人类操作
- 避免设置过大的随机范围影响业务逻辑

### 8.8 装饰参数

装饰参数用于改变节点的执行行为：

\| 参数     | 说明                      | 适用节点类型       |
\| 启用状态  | 是否启用该节点（默认启用）     | 所有节点           |
\| 结果取反  | 勾选后，成功变为失败，失败变为成功 | 所有节点           |
\| 失败重试  | 设置节点失败后的重试次数       | 所有节点           |
\| 重复次数  | 设置节点成功后的重复执行次数    | 所有节点           |
\| 重复间隔  | 重复执行之间的间隔时间（毫秒）  | 所有节点           |
\| 超时时间  | 超时时间（毫秒）             | 所有节点           |
\| 检测间隔  | 条件节点的检测间隔（毫秒）      | 仅条件节点          |

## 9. 黑板系统

### 9.1 什么是黑板？

黑板是节点间通信的核心机制，采用观察者模式，允许节点间解耦通信。

### 9.2 内置变量

| 变量名                       | 说明                       |
| ------------------------- | ------------------------ |
| last\_detection\_position | 最近一次检测到的位置坐标             |
| last\_detection\_x        | 最近一次检测位置的 x 坐标分量（自动拆分写入） |
| last\_detection\_y        | 最近一次检测位置的 y 坐标分量（自动拆分写入） |
| last\_number\_value       | 最近一次识别的数字值               |
| last\_extracted\_text     | 最近一次文本提取的内容              |
| last\_input\_text         | 最近一次文本输入的内容（需开启保存）       |
| execution\_count          | 执行计数                     |

### 9.3 使用方式

**写入变量**：

- 条件节点检测成功后自动写入位置
- 设变量节点主动设置变量值

**读取变量**：

- 点击节点使用"点击最近检测点"功能
- 变量判断节点判断变量值

### 9.4 典型通信场景

**场景：OCR检测 → 点击检测位置**

```

     │                                    │ 3. blackboard.get()
```

### 10.5 多 Tab 并行编辑与运行

**功能概述**：

多 Tab 并行功能支持同时打开多个行为树项目，每个项目在独立的 Tab 中运行，拥有独立的画布、引擎、黑板和撤销/重做历史。

**界面布局**：

```
┌─────────────────────────────────────────────────────────────────────────┐
│                              工具栏                                      │
│  [新建][打开][保存][导出] │ [撤销][重做][清空] │ [▶运行][⏹停止] │ [重置] │
├─────────────────────────────────────────────────────────────────────────┤
│  [Tab: 项目A] [Tab: 项目B] [Tab: 项目C] [+]                              │
├────────────┬───────────────────────────────────────────┬────────────────┤
│            │                                           │                │
│   节点面板  │              画布区域                      │    属性面板    │
│            │                                           │                │
│            │                                           │                │
└────────────┴───────────────────────────────────────────┴────────────────┘
```

**Tab 操作说明**：

| 操作        | 说明          |
| --------- | ----------- |
| 点击 Tab    | 切换到对应项目     |
| 点击 "+" 按钮 | 导入新项目到新 Tab |
| 双击 Tab    | 重命名 Tab 名称  |
| 右键 Tab    | 关闭该 Tab     |

**运行模式**：

| 模式   | 操作                | 说明                  |
| ---- | ----------------- | ------------------- |
| 全部运行 | 点击工具栏【运行】按钮或按 F10 | 同时运行所有已加载 Tab 中的行为树 |
| 全部停止 | 点击工具栏【停止】按钮或按 F12 | 同时停止所有正在运行的行为树      |
| 单个运行 | 点击 Tab 内的 ▶ 按钮    | 仅运行当前 Tab 的行为树      |
| 单个停止 | 点击 Tab 内的 ⏹ 按钮    | 仅停止当前 Tab 的行为树      |

**Tab 独立性**：

每个 Tab 拥有完全独立的资源：

| 资源    | 说明         |
| ----- | ---------- |
| 画布    | 独立的节点编辑画布  |
| 引擎    | 独立的行为树执行引擎 |
| 黑板    | 独立的数据存储空间  |
| 撤销/重做 | 独立的操作历史    |
| 自动保存  | 独立的自动保存管理  |

**使用步骤**：

**步骤1：导入多个项目**

1. 点击 Tab 栏右侧的【+】按钮
2. 选择要导入的项目文件夹
3. 项目自动在新 Tab 中打开
4. 重复以上步骤导入更多项目

**步骤2：切换编辑**

- 点击不同的 Tab 切换项目
- 每个项目的编辑操作完全独立
- 撤销/重做仅影响当前 Tab

**步骤3：并行运行**

- 点击工具栏【运行】按钮同时运行所有项目
- 或点击单个 Tab 内的 ▶ 按钮运行单个项目
- 观察各项目的运行状态和日志

**步骤4：保存与关闭**

- 切换 Tab 时自动保存当前项目
- 关闭 Tab 时检查未保存修改并提示
- 关闭应用时自动保存 Tab 列表

**Tab 持久化**：

- 关闭应用时自动保存已打开的 Tab 列表
- 下次启动应用时自动恢复上次打开的 Tab
- 恢复后自动选中上次活动的 Tab

**注意事项**：

1. 每个项目的黑板数据完全隔离，不会相互影响
2. 运行多个项目时会占用更多系统资源
3. 建议根据任务复杂度合理控制同时运行的项目数量
4. 关闭未保存的 Tab 时会提示保存，避免数据丢失

## 11. 项目管理

### 11.1 项目文件夹结构

每个行为树项目都是一个独立的文件夹，包含完整的行为树定义和所有关联资源：

```
MyProject/                    # 项目根文件夹

│
├── images/                   # 图像资源
│   ├── templates/            # 模板图片
│   └── screenshots/          # 截图文件
│
├── scripts/                  # 脚本资源
│   ├── script/               # 脚本文件
│   └── code/                 # 代码文件
│
├── audio/                    # 音频资源
│   └── alarms/               # 报警音效
│
├── data/                     # 数据资源
│   └── config/               # 配置文件
│
├── cache/                    # 缓存文件夹
│
└── docs/                     # 文档文件夹
```

### 11.2 创建新项目

1. 点击工具栏【新建项目】按钮
2. 填写项目信息：
   - 项目名称
   - 保存位置
   - 项目描述（可选）

1. 点击工具栏【打开项目】按钮
2. 选择项目文件夹

1. 在节点属性面板中选择资源文件

| 节点属性           | 资源类型   | 目标目录              |

- 临时文件
- 备份文件（tree\_backup\_\*）

- 备份文件命名：`tree_backup_YYYYMMDD_HHMMSS.json`

## 14. 常见问题

### Q1: 如何实现循环执行？

**A**: 设置开始或组合节点的"重复次数"为 -1 即可实现无限循环。

### Q2: 点击位置不正确怎么办？

**A**:

1. 检查检测区域是否正确
2. 确保勾选了"点击最近检测点"
3. 检测节点会保存绝对坐标，点击节点会使用该坐标

### Q3: OCR识别不准确怎么办？

**A**:

1. 确保检测区域只包含需要识别的文字
2. 尝试更换 OCR 语言设置
3. 中文识别已针对中文特点优化

### Q4: 图像匹配失败怎么办？

**A**:

1. 降低匹配阈值（如从 90 降到 80）
2. 确保模板图像清晰
3. 检查检测区域是否正确

### Q5: 如何使用"点击最近检测点"？

**A**:

1. 在条件节点（OCR/图像/颜色）中检测目标
2. 检测成功后位置会自动保存到黑板
3. 在点击节点中勾选"点击最近检测点"
4. 点击时会使用最近一次检测到的位置

### Q6: 如何实现长按操作？

**A**: 在点击节点中设置：

- 动作：press
- 按住时长：500（毫秒）

### Q7: 如何实现拖拽操作？

**A**: 使用鼠标移动节点的拖拽功能：

1. 设置移动类型=拖拽
2. 设置位置为拖拽起点
3. 设置拖拽终点为目标位置
4. 设置拖拽时长（建议 500-1000 毫秒）

### Q8: 如何实现无限点击？

**A**: 在点击节点中设置：

- 点击次数：-1
- 点击间隔：100（毫秒）
- 节点会持续点击，直到按 Escape 键停止

### Q9: 节点执行顺序如何确定？

**A**:

- 连接线上会显示序号（1、2、3...）
- 按序号顺序执行
- 如需调整，删除连接线后重新按期望顺序连接

### Q10: 如何分享项目？

**A**:

1. 点击【导出】按钮导出为 ZIP 文件
2. 将 ZIP 文件分享给其他用户
   1. 其他用户点击【导入】按钮，选中 ZIP 文件即可打开项目
